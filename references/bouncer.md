# Bouncer

## Defaults

- Install with `node ace add @adonisjs/bouncer`.
- Keep `initialize_bouncer_middleware` registered in `start/kernel.ts`.
- Prefer policies for resource authorization.
- Use abilities only for simple cross-cutting checks.

## Policy Shape

- Create one policy per resource with `node ace make:policy <resource>`.
- Default policy method set:
  - `viewList`
  - `view`
  - `create`
  - `update`
  - `delete`
  - explicit verbs like `publish`, `archive`, `restore`

- This method set is a skill convention to make policies predictable. Bouncer itself allows arbitrary method names.

## Controller Rule

- Load the resource.
- Run the policy check immediately.
- Abort before the service call if unauthorized.

```ts
import PostPolicy from '#policies/post_policy'

if (await bouncer.with(PostPolicy).denies('update', post)) {
  return response.forbidden()
}
```

## Edge and Pre-Registration

- Keep abilities in `app/abilities/main.ts`.
- Keep policy registration in the policy collection generated by the package.
- If the repo uses Edge authorization tags, keep abilities and policies pre-registered so `@can` stays clean.

## Access Tokens

- If the application uses access tokens, authorization may depend on both the user and `currentAccessToken` abilities.
- Keep that logic in Bouncer, not scattered across controllers.
